02 — الـ CSS Keyframes

تشريح الحركة خطوة بخطوة

الأنيميشن مقسّم على 4 نقاط رئيسية (keyframes) — كل نقطة تحدد حجم وشفافية الـ card في تلك اللحظة

0%
scale(0)
غير مرئي
60%
scale(1.12)
أكبر من الطبيعي
80%
scale(0.95)
ارتداد خفيف
100%
scale(1)
استقرار نهائي
اللحظة transform opacity ما يحدث
0% scale(0) 0 البداية — الـ card غير موجودة بصرياً
60% scale(1.12) 1 تكبر بسرعة وتتجاوز حجمها الطبيعي بـ 12% — هذا يعطي إحساس الفقاعة
80% scale(0.95) 1 ترتد قليلاً للخلف — يحاكي قانون الفيزياء (bounce)
100% scale(1) 1 تستقر على حجمها الطبيعي — نهاية الحركة
CSS — @keyframes
/* الخطوة 1: إخفاء كل الـ cards مبدئياً */
.skill-card {
  opacity: 0;
  transform: scale(0);
  transform-origin: center center;
}

/* الخطوة 2: تعريف الحركة */
@keyframes bubblePop {
  0%   { transform: scale(0);    opacity: 0; }
  60%  { transform: scale(1.12); opacity: 1; }
  80%  { transform: scale(0.95); }
  100% { transform: scale(1);    opacity: 1; }
}

/* الخطوة 3: class تُشغّل الحركة عند إضافتها */
.bubble-animate {
  animation: bubblePop 0.45s
    cubic-bezier(0.36, 0.07, 0.19, 0.97)
    forwards;
}
03 — الـ JavaScript

التدرج في الظهور — card بعد card

السحر يكمن في استخدام setTimeout مع مضاعفة الـ index — كل card تنتظر مدة أطول من التي قبلها

معادلة تأخير كل card
delay = index × interval
مثال: interval = 120ms
card[0] → 0 × 120 = 0ms  |  card[1] → 1 × 120 = 120ms  |  card[5] → 5 × 120 = 600ms
JavaScript — الدالة الرئيسية
function animateSkillCards() {
  const cards    = document.querySelectorAll('.skill-card');
  const interval = 120; // المسافة الزمنية بين كل card (ms)

  cards.forEach((card, index) => {
    const delay = index * interval; // ← المعادلة هنا

    setTimeout(() => {
      card.classList.add('bubble-animate'); // تشغيل الحركة
    }, delay);
  });
}
JavaScript — الاستدعاء الصحيح (مع fetch)
// إذا كانت الـ cards تُبنى من JSON
fetch('data/skills.json')
  .then(res => res.json())
  .then(data => {
    const skills = data.skills || data;
    const container = document.getElementById('skills-container');
    container.innerHTML = '';

    skills.forEach(skill => {
      // ... بناء الـ card وإضافتها للـ DOM ...
      container.appendChild(card);
    });

    animateSkillCards(); // ← بعد اكتمال البناء مباشرة

// إذا كانت الـ cards موجودة في HTML مباشرة
document.addEventListener('DOMContentLoaded', () => {
  animateSkillCards();
});
1

لماذا نضيف class بدلاً من تحديد الـ style مباشرة?

الـ class قابلة للإزالة وإعادة الإضافة — هذا يسمح لنا بعمل Replay. لو غيّرنا الـ style مباشرة ما قدرنا نعيد الأنيميشن بسهولة.

2

أهمية كلمة forwards

بدونها، بعد انتهاء الأنيميشن ترجع الـ card لحالتها الأولى (opacity: 0, scale: 0) وتختفي. الـ forwards تجعلها تحافظ على آخر حالة (scale: 1, opacity: 1).

3

لماذا index * interval وليس قيمة ثابتة?

لو استخدمنا قيمة ثابتة لكل card، ستظهر كلها في نفس الوقت. الضرب في الـ index يضمن أن كل card تنتظر مدة أطول من التي قبلها بمقدار ثابت (الـ interval).

05 — الـ Cubic Bezier

سر طبيعية الحركة

الـ easing function هي ما تجعل الحركة تبدو طبيعية وليست آلية — نستخدم cubic-bezier(0.36, 0.07, 0.19, 0.97)

قيم نقاط التحكم

P1x
0.36
P1y
0.07
P2x
0.19
P2y
0.97

هذه القيم تجعل الحركة تبدأ ببطء، تتسارع بسرعة في المنتصف، ثم تنتهي بسرعة — يشبه ضغطة زر حقيقية

نوع الـ easing القيمة الإحساس مناسب لـ
مخصص (مستخدم) cubic-bezier(0.36,0.07,0.19,0.97) فقاعة طبيعية Bubble Pop
ease-out cubic-bezier(0,0,0.2,1) يبدأ بسرعة ويهدأ Slide In
ease-in-out cubic-bezier(0.4,0,0.6,1) ناعم من الطرفين Fade
linear linear ميكانيكي بدون روح Progress bars

💡 نصيحة أداء

استخدم دائماً transform و opacity في الأنيميشن وليس width أو height أو margin. المتصفح يعالج الـ transform والـ opacity على الـ GPU مباشرة (compositor layer) مما يجعلها سلسة جداً حتى مع 60 fps. الخصائص الأخرى تسبب reflow أو repaint وهذا يؤدي إلى تقطع ملحوظ.
