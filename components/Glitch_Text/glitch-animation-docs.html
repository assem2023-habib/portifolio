دليل تقني كامل يشرح بناء حركة Glitch تفاعلية مرتبطة بسرعة الماوس، تعمل على مستوى كل حرف بشكل مستقل، مع نظام recovery ديناميكي. مكتوب لكل من المطور البشري والـ AI Agent.

■ المفهوم

ما الذي يجعل هذا الـ Glitch مختلفاً؟

معظم تأثيرات الـ Glitch المتاحة تعمل على النص كاملاً، وتبدأ تلقائياً، وتتكرر بشكل آلي. هذا النظام مختلف جذرياً في ثلاثة محاور:

■ Per-Character

كل حرف يعمل كـ unit مستقل تماماً. له slices خاصة، وRGB channels منفصلة، ودورة حياة خاصة. الحرف يتأثر فقط عندما يدخل الماوس مساحته.

■ Velocity-Driven

شدة التأثير تساوي شدة السرعة. الماوس البطيء ينتج تشويشاً خفيفاً. الماوس السريع ينتج فوضى كاملة. لا يوجد تأثير ثابت.

■ Direction-Aware

الـ slices تتحرك في نفس اتجاه الماوس. القناة الحمراء تذهب مع الاتجاه، السماوية عكسه. الحركة تشعر بأنها ناتجة عن الاحتكاك الرقمي.

■ Dynamic Recovery

العودة ليست ثابتة. السرعة العالية تنتج elastic overshoot — الحرف يتجاوز الوضع الأصلي قليلاً ثم يرتد. السرعة المنخفضة تنتج ease-out ناعم.

■ البنية

البنية المعمارية للكود

كل حرف يُبنى كـ component مستقل يحتوي على طبقات متعددة تُكدَّس فوق بعضها. فهم هذه الطبقات ضروري قبل أي تعديل.

HTML Structure — كل حرف 
<!-- هذا الـ HTML يُبنى ديناميكياً بالـ JS لكل حرف -->
<div class="g-char-wrap">           <!-- الحاوية الرئيسية -->

  <!-- طبقة RGB الحمراء — خلف النص الأصلي -->
  <span class="g-char-rgb red">N</span>

  <!-- طبقة RGB السماوية — خلف النص الأصلي -->
  <span class="g-char-rgb cyn">N</span>

  <!-- النص الأصلي — يختفي أثناء الـ glitch -->
  <span class="g-char-base">N</span>

  <!-- حاوية الـ slices — تُملأ بالـ JS -->
  <div class="g-char-slices">
    <!-- slice مثال -->
    <div class="g-char-slice" style="top:0;height:8px">
      <div class="g-char-slice-inner" style="top:0px">N</div>
    </div>
    <!-- ... المزيد من الـ slices -->
  </div>

</div>

لماذا هذه البنية تحديداً؟

g-char-base موجود دائماً ويحدد أبعاد الحرف. عند تشغيل الـ glitch نجعله opacity:0 فقط ولا نحذفه — هذا يمنع الـ layout shifts.

g-char-slices تحتوي على overflow:hidden مما يجعل كل slice تعرض فقط الجزء الخاص بها من النص عبر clipping تلقائي. هذا هو جوهر تأثير الـ slice.

g-char-slice-inner يحتوي على نفس النص لكن top تساوي سالب موضع الـ slice، مما يجعل النص يظهر في الموضع الصحيح داخل كل شريحة.

◈

مخطط الـ z-index layers

/* من الأسفل للأعلى */ z-index: 3 → RGB ghost layers /* mix-blend-mode:screen */ z-index: 5 → Base text /* النص الأصلي */ z-index: 6 → Slice bands /* يغطي Base أثناء glitch */ z-index: 7 → Pixel fragments /* فوق كل شيء */

Phase 1 — Clean State

الحالة الافتراضية للحرف قبل أي تفاعل. هذه هي نقطة البداية والنهاية في كل دورة.

PH.01 Clean State — الحالة النظيفة ∞ until hover

ماذا يحدث: النص ظاهر بشكل حاد وكامل. لا حركة، لا تشويش، لا تأثيرات.

ما هو موجود في DOM لكنه مخفي: طبقتا RGB (opacity:0)، حاوية الـ slices (فارغة)، حاوية الـ fragments (فارغة).

السبب: الحفاظ على DOM ثابت يمنع reflow عند بدء الحركة، وهو أهم من الأداء اللحظي.

Phase 2 — Digital Corruption

الإشارة الأولى للتشويش. النص يُقسَّم إلى شرائح أفقية تتحرك بشكل عشوائي.

PH.02 Digital Corruption — التشويش الرقمي 60ms – 160ms (حسب السرعة)

الاسم التقني: Horizontal Slice Displacement

ماذا يحدث: ارتفاع الحرف يُقسَّم إلى شرائح أفقية. كل شريحة تعرض جزءاً من النص وتتحرك أفقياً بشكل مستقل عن الأخريات.

🟢 سرعة منخفضة

8–12 شريحة، ارتفاع 8–18px، إزاحة 3–8px، تُحدَّث كل 18ms

🔴 سرعة عالية

40–50 شريحة، ارتفاع 2–6px، إزاحة 15–42px، تُحدَّث كل 8ms

JavaScript — buildSlices()
function buildSlices(s, intensity) {
  s.slicesCon.innerHTML = '';
  s.slices = [];
  const H = s.base.offsetHeight;

  /* عدد الشرائح: يزيد مع السرعة */
  const maxSlices = Math.round(lerp(8, 50, intensity));

  /* ارتفاع الشريحة: يقل مع السرعة (تقطيع أدق) */
  const minH = Math.round(lerp(8, 2, intensity));
  const maxH = Math.round(lerp(18, 6, intensity));

  let y = 0;
  while (y < H && s.slices.length < maxSlices) {
    const h = clamp(rndInt(minH, maxH), 2, H - y);

    /* كل شريحة = div يقطع بـ overflow:hidden */
    const el = document.createElement('div');
    el.className = 'g-char-slice';
    el.style.top = y + 'px';
    el.style.height = h + 'px';

    /* النص داخل الشريحة — يُعوَّض لأعلى لمحاذاة الحرف */
    const inner = document.createElement('div');
    inner.className = 'g-char-slice-inner';
    inner.textContent = s.letter;
    inner.style.top = (-y) + 'px'; /* التعويض الحاسم */

    el.appendChild(inner);
    s.slicesCon.appendChild(el);
    s.slices.push({ el, cx: 0, velX: 0 });
    y += h;
  }
}

لماذا top: (-y) مهم جداً؟

إذا وضعنا الحرف داخل div بارتفاع 8px فقط، سيظهر الجزء العلوي من الحرف فقط في كل شريحة. بإضافة top: -y نعوّض الموضع لأعلى بمقدار بداية الشريحة، فيظهر الجزء الصحيح من الحرف في كل شريحة. هذا هو المبدأ الأساسي لـ CSS Clipping بدون clip-path.

PH.03 RGB Channel Split — انقسام القنوات اللونية يبدأ بعد 35ms — يستمر حتى Recovery

المفهوم: نسخ من النص بلونين مختلفين (أحمر + سماوي) تُزاح في اتجاهين معاكسين. mix-blend-mode:screen يجعلها مرئية فقط على الخلفيات الداكنة ويمنع البياض الكامل.

القناة الحمراء (#ff1a3a)

تتحرك في نفس اتجاه الماوس. إزاحة +1 إلى +8px. تومض بشكل مستقل.

القناة السماوية (#00e5ff)

تتحرك في الاتجاه المعاكس للماوس. إزاحة -1 إلى -8px. تومض بشكل مستقل.

JavaScript — rgbFrame()
function rgbFrame(s, intensity, dirSign) {
  /* الإزاحة القصوى: 1px عند بطء شديد → 8px عند سرعة قصوى */
  const maxOff = lerp(1, 8, intensity);

  /* الشفافية: 8% عند بطء → 50% عند سرعة */
  const maxOpa = lerp(0.08, 0.50, intensity);

  /* احتمال الظهور: يزيد مع السرعة */
  const flickerChance = lerp(0.55, 0.20, intensity);

  /* القناة الحمراء → اتجاه الماوس */
  s.rgbR.style.transform = `translateX(${dirSign * rnd(1, maxOff)}px)`;
  s.rgbR.style.opacity = Math.random() > flickerChance
    ? String(rnd(0.06, maxOpa)) : '0';

  /* القناة السماوية → عكس اتجاه الماوس */
  s.rgbC.style.transform = `translateX(${-dirSign * rnd(1, maxOff)}px)`;
  s.rgbC.style.opacity = Math.random() > flickerChance
    ? String(rnd(0.06, maxOpa)) : '0';
}

تنبيه: لا تستخدم CSS transitions مع القنوات اللونية. التحديث يجب أن يكون فورياً في كل frame لتحقيق تأثير الوميض السريع. أي transition ستحوّل الوميض إلى fade بطيء.

Phase 4 — Distortion Peak

ذروة التشويش. الشدة ترتفع حتى نقطة الذروة ثم تبدأ بالتراجع تدريجياً.

PH.04 Distortion Peak — ذروة التشويش 80ms – 280ms (حسب السرعة)

ما يجعله مختلفاً عن Phase 2: في Phase 2 الشدة ثابتة (~0.5). في Phase 4 الشدة تتغير مع الوقت — ترتفع من 0.55 إلى 1.0 عند 42% من المدة، ثم تنخفض تدريجياً إلى 0.25. هذه هي "الإيقاعية" التي تجعل التأثير يشعر بأنه حي.

JavaScript — loopP4 intensity curve
const p4Peak = p4Start + P4_DUR * 0.42; /* الذروة عند 42% */

/* داخل الـ animation loop */
const ratio = (now - p4Start) / P4_DUR;
const intensity =
  now < p4Peak
  /* صعود: من 0.55*intensity إلى intensity */
  ? lerp(0.55 * baseIntensity, baseIntensity,
          (now - p4Start) / (p4Peak - p4Start))

  /* هبوط: من intensity إلى intensity*0.25 */
  : lerp(baseIntensity, baseIntensity * 0.25,
          (now - p4Peak) / (p4End - p4Peak));

Phase 5 — Pixel Fragmentation

جزيئات رقمية تتطاير من الحرف. تظهر فقط عند السرعات العالية (intensity > 0.45).

PH.05 Pixel Fragmentation — تشرذم البكسل 60ms – 180ms per fragment

المفهوم: مستطيلات صغيرة لونها أبيض (نفس لون النص) تُنشأ عشوائياً على مساحة الحرف وتتطاير في اتجاه الماوس بتسارع متناقص (ease-out)، ثم تتلاشى حتى opacity:0.

عدد الجزيئات: lerp(3, 18, intensity) — من 3 قطع ببطء إلى 18 قطعة بسرعة قصوى.

أبعاد كل جزيئة: عرض 3–22px، ارتفاع 1–5px. مستطيلات أفقية لتناسب مظهر البيانات الرقمية.

JavaScript — spawnFragments()
function spawnFragments(s, intensity, dirSign) {
  const count = Math.round(lerp(3, 18, intensity));

  for (let i = 0; i < count; i++) {
    const fw = rndInt(3, Math.round(lerp(8, 22, intensity)));
    const fh = rndInt(1, 5);
    const fx = rnd(W*0.02, W*0.98 - fw);
    const fy = rnd(0, H - fh);

    /* اتجاه الطيران = اتجاه الماوس، مسافة حسب السرعة */
    const jumpX = dirSign * rnd(8, 55 * intensity);
    const dur   = rnd(60, 180 * intensity);

    /* كل جزيئة لها دورة حياة مستقلة */
    setTimeout(() => {
      (function tick(now) {
        const p = clamp((now - t0) / dur, 0, 1);
        const e = easeOutCubic(p);           /* تباطؤ طبيعي */
        el.style.transform = `translateX(${jumpX * e}px)`;
        el.style.opacity = String(1 - e);    /* تلاشٍ متزامن */
        if (p < 1) requestAnimationFrame(tick);
        else fcon.remove();                  /* تنظيف DOM */
      })(performance.now());
    }, rnd(0, 60)); /* تأخير عشوائي لكل جزيئة */
  }
}

Phase 6 — Dynamic Recovery

المرحلة الأكثر دقة في النظام. طريقة العودة تختلف جذرياً حسب السرعة الأصلية.

PH.06 Dynamic Recovery — العودة الديناميكية 80ms – 320ms (حسب السرعة)

المفهوم: كل slice تعود إلى موضع X=0 ولكن بمسار مختلف حسب شدة التشويش.

مستوى السرعة نوع الـ Easing المدة السلوك
LOW easeOutCubic ~80ms سريع وناعم. مباشر إلى الصفر.
MED easeOutQuint ~180ms أبطأ في البداية، توقف تدريجي.
HIGH easeOutQuint ~250ms بطيء جداً في النهاية، ثقيل.
EXTREME Elastic Spring ~320ms يتجاوز الصفر (overshoot) ثم يرتد.
JavaScript — Recovery Easing Selection
/* اختيار نوع الـ easing حسب شدة السرعة */
let e;
if (intensity < 0.40) {
  /* LOW: ناعم وسريع */
  e = easeOutCubic(p);   /* 1 - (1-p)³ */

} else if (intensity < 0.75) {
  /* MED-HIGH: أبطأ وأثقل */
  e = easeOutQuint(p);   /* 1 - (1-p)⁵ */

} else {
  /* EXTREME: elastic spring — يتجاوز ثم يرتد */
  const c4 = (2 * Math.PI) / 3;
  e = p === 0 ? 0 : p === 1 ? 1 :
    Math.pow(2, -10*p) * Math.sin((p*10 - 0.75) * c4) + 1;
}

/* كل slice تعود من موضعها الحالي إلى الصفر */
s.slices.forEach((sl, i) => {
  const pos = lerp(startX[i], 0, e);
  sl.el.style.transform = `translateX(${pos}px)`;

■ نظام السرعة

كيف نقيس سرعة الماوس

السرعة الخام من أحداث mousemove متقلبة جداً. نستخدم exponential smoothing لتحويلها إلى سرعة ناعمة وقابلة للاستخدام.

JavaScript — Velocity Tracker
let smoothVX = 0, smoothVY = 0;
let lastMX = 0, lastMY = 0, lastT = 0;

window.addEventListener('mousemove', e => {
  const now = performance.now();
  const dt  = Math.max(now - lastT, 1); /* منع القسمة على صفر */

  /* السرعة الخام = التغيير في الموضع ÷ الزمن × معامل تطبيع */
  const dvx = (e.clientX - lastMX) / dt * 16;
  const dvy = (e.clientY - lastMY) / dt * 16;

  /* Exponential smoothing — α=0.40 — يمزج الجديد مع القديم */
  smoothVX = lerp(smoothVX, dvx, 0.40);
  smoothVY = lerp(smoothVY, dvy, 0.40);

  lastMX = e.clientX;
  lastMY = e.clientY;
  lastT  = now;
});

/* تناقص تدريجي عند توقف الماوس */
setInterval(() => {
  smoothVX *= 0.88;
  smoothVY *= 0.88;
}, 30);
rawSpeed = √(smoothVX² + smoothVY²) intensity = clamp((rawSpeed - 1.5) / 26.5, 0.0, 1.0) dirSign = smoothVX >= 0 ? +1 : -1 /* أمثلة عملية */ rawSpeed = 0 → intensity = 0.00 ← لا تأثير rawSpeed = 5 → intensity = 0.13 ← تشويش خفيف rawSpeed = 14 → intensity = 0.47 ← تشويش متوسط rawSpeed = 28 → intensity = 1.00 ← تشويش أقصى

لماذا نطرح 1.5 من rawSpeed؟

هذه هي الـ dead zone. الحركة الطبيعية للماوس حتى عند التحريك البطيء جداً تنتج قيماً صغيرة من الضجيج. طرح 1.5 يمنع ظهور تأثير خفيف عند الحركة البطيئة جداً التي يجب ألا تنتج أي تأثير.

الـ Easing Functions المستخدمة

JavaScript — جميع منحنيات الحركة
/* ─── الدوال الأساسية ─── */

/* Ease Out Cubic — ناعم وطبيعي، للسرعات المنخفضة */
const easeOutCubic  = t => 1 - Math.pow(1 - t, 3);

/* Ease Out Quint — أثقل وأبطأ في النهاية، للسرعات المتوسطة */
const easeOutQuint  = t => 1 - Math.pow(1 - t, 5);

/* Elastic Spring — يتجاوز الهدف ثم يرتد، للسرعات العالية */
const elasticOut = t => {
  if (t === 0) return 0;
  if (t === 1) return 1;
  const c4 = (2 * Math.PI) / 3;
  return Math.pow(2, -10 * t) *
         Math.sin((t * 10 - 0.75) * c4) + 1;
};

/* Ease Out Quad — للشرائح أثناء الـ glitch */
const easeOutQuad   = t => 1 - (1 - t) * (1 - t);

/* Linear Interpolation — الأساس لكل شيء */
const lerp = (a, b, t) => a + (b - a) * t;

function triggerChar(s) {
  if (s.running) return; /* منع التشغيل المزدوج */
  s.running = true;

  /* ❶ قراءة السرعة والاتجاه لحظة الدخول */
  const intensity = getIntensity();
  const dirSign   = getDirSign();

  /* ❷ حساب توقيتات كل مرحلة */
  const P2_DUR   = Math.round(lerp(60,  160, intensity));
  const P3_START = Math.round(lerp(20,  40,  intensity));
  const P4_DUR   = Math.round(lerp(80,  280, intensity));
  const P4_START = P2_DUR + 10;
  const REC_DUR  = Math.round(lerp(80,  320, intensity));
  const REC_START = P4_START + P4_DUR + 30;

  /* ❸ بناء الـ slices وتفعيلها */
  buildSlices(s, intensity);
  activateSlices(s);

  /* ❹ Phase 2 loop */
  startPhase2Loop(s, intensity, dirSign, P2_DUR);

  /* ❺ Phase 3 — بعد P3_START */
  setTimeout(() => startRGBLoop(s, intensity, dirSign), P3_START);

  /* ❻ Phase 4 — بعد P4_START */
  setTimeout(() => startPhase4Loop(s, intensity, dirSign, P4_DUR), P4_START);

  /* ❼ Phase 5 — fragments (فقط عند سرعة كافية) */
  if (intensity > 0.45) {
    setTimeout(() => spawnFragments(s, intensity, dirSign), P4_START + 50);
  }

  /* ❽ Phase 6 — Recovery */
  setTimeout(() => startRecovery(s, intensity, REC_DUR), REC_START);
}
