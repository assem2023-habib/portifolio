## 📖 نظرة عامة شاملة

**ميزانيتي** ليس مجرد تطبيق لتسجيل المصاريف، بل هو نظام مالي متكامل يعيد تعريف علاقتك بأموالك. صُمم التطبيق باستخدام أقوى الممارسات البرمجية العالمية ليوفر أداة تجمع بين **الجمالية البصرية** و**الدقة المحاسبية**.

### 🎯 الفائدة الحقيقية من المشروع
يهدف "ميزانيتي" إلى حل "الفوضى المالية" التي يواجهها الأفراد من خلال:
- 💡 **الوعي المالي الفوري:** رؤية واضحة لأين تذهب كل ليرة أو دولار.
- 🛑 **كبح الإنفاق الاندفاعي:** عبر نظام ميزانيات ذكي يحذرك قبل فوات الأوان.
- 📑 **التوثيق الرقمي:** الاحتفاظ بصور الفواتير والإيصالات مرتبطة بمعاملاتها للأبد.
- 🏢 **تعدد المحافظ:** الفصل التام بين أموال العمل، الأموال الشخصية، والادخار.

## ✨ المميزات الرئيسية (بالتفصيل)

### 🧮 نظام المعاملات والتحويلات
- **إدارة الدخل والمصروفات:** تسجيل فوري مع دعم للكسور المالية الدقيقة.
- **التحويلات الداخلية:** نقل الأموال بين الحسابات (مثلاً من الحساب البنكي إلى الكاش) مع تحديث تلقائي للأرصدة في كلا الحسابين.
- **الملاحظات المتقدمة:** دعم كتابة ملاحظات طويلة وتاريخية لكل معاملة.
- **التوثيق البصري:** دمج كاميرا الجوال لالتقاط صور الإيصالات وتخزين مساراتها محلياً.

### 📊 الإحصائيات والذكاء المالي
- **محرك fl_chart:** رسوم بيانية تفاعلية تظهر توزع الإنفاق حسب الفئات.
- **تحليل الاتجاهات:** مقارنة الإنفاق في الشهر الحالي مقابل الأشهر السابقة.
- **رؤى الرصيد:** مراقبة نمو أو تراجع صافي ثروتك (Net Worth) عبر الزمن.

### ⚙️ تخصيص الأنظمة
- **نظام الفئات (Categories):** أكثر من 20 فئة افتراضية مع إمكانية إنشاء فئات لا نهائية بألوان مخصصة من لوحة ألوان Hex.
- **تخصيص الحسابات:** دعم أنواع (Cash, Bank, Credit Card, PayPal) مع أيقونات تمييزية.

## 🏗️ هندسة بناء المشروع (System Architecture)

يتبع المشروع هيكلية **Clean Architecture** مع تقسيم **Feature-First** داخل طبقة العرض، مما يضمن أقصى درجات الفصل بين المهام:

### 1️⃣ طبقة النطاق (Domain Layer) - "قلب النظام"
- **Entities:** تمثل الكيانات الصرفة التي لا تعتمد على أي مكتبات خارجية.
- **Value Objects:** (الابتكار الأهم) حيث لا يتم التعامل مع المبالغ كأرقام `double` عادية، بل ككائنات `Money` تحمي نفسها من القيم السالبة أو غير المنطقية.
- **Usecases:** تحتوي على 13 حالة استخدام (مثل `CreateAccount`, `DeleteTransaction`) تنفذ وظيفة واحدة فقط (Single Responsibility).

### 2️⃣ طبقة البيانات (Data Layer) - "مخزن الحقيقة"
- **Drift (Moor):** قاعدة بيانات علائقية آمنة من حيث النوع (Type-safe).
- **DAOs (Data Access Objects):**
    - `AccountsDao`: مسؤول عن عمليات الحسابات والأرصدة.
    - `TransactionsDao`: مسؤول عن المعاملات والتحويلات.
    - `CategoriesDao`: مسؤول عن تصنيفات البيانات.
    - `BudgetsDao`: مسؤول عن مراقبة الميزانيات.
- **Mappers:** محولات ذكية تقوم بتحويل `Drift Data Classes` إلى `Domain Entities` والعكس.

### 3️⃣ طبقة التطبيق (Application Layer)
- **Riverpod Providers:** نظام حقن تبعيات متطور يدير دورة حياة الكائنات.
- **State Management:** استخدام `Notifier` و `AsyncNotifier` للتعامل مع العمليات غير المتزامنة لضمان استجابة الواجهة.

## 🧠 المنطق الرياضي والحسابي (Mathematical Core)

### ⚖️ خوارزمية حساب الرصيد
لا يعتمد التطبيق على قيمة مخزنة ثابتة للرصيد، بل يقوم بـ:
1. حساب الرصيد الافتتاحي للحساب.
2. جمع كل المعاملات من نوع (Income).
3. طرح كل المعاملات من نوع (Expense).
4. إضافة/طرح التحويلات الصادرة والواردة.
5. النتيجة تضمن دقة بنسبة 100% حتى لو تم تعديل معاملة قديمة.

### 📉 منطق الميزانيات (Budget Calculation)
يتم حساب حالة الميزانية عبر المعادلة:
$$Status = (Consumed \div Limit) \times 100$$
مع مراعاة النطاق الزمني (يومي، أسبوعي، شهري) وربطه بتاريخ المعاملات المسجلة.

## 🔐 نظام الأمان والتحقق (Validation Logic)

قبل أن تصل أي معلومة لقاعدة البيانات، تمر عبر **11 دالة تحقق** صارمة:
- **Validators.validateAmount:** يمنع تسجيل مبالغ صفرية أو سالبة.
- **Validators.validateAccountActive:** يمنع إجراء معاملات على حسابات مغلقة أو مؤرشفة.
- **Validators.validateDate:** يضمن عدم تسجيل معاملات في تواريخ مستقبلية غير منطقية (حسب إعدادات المستخدم).

### المكتبات الأساسية (The Main Packages)
| الحزمة | الدور الوظيفي |
| :--- | :--- |
| `flutter_riverpod` | العمود الفقري لإدارة الحالة وحقن التبعيات. |

## 📁 هيكل المجلدات التفصيلي (Folder Structure)

```text
lib/
├── core/
│   ├── constants/        # ثوابت الألوان، الأيقونات، والحدود المالية
│   ├── errors/           # تعريف أنواع الفشل (Validation, DB, Auth)
│   ├── logging/          # DbLogger لتتبع أداء الاستعلامات
│   └── utils/            # Result Pattern و Extension Methods
├── domain/
│   ├── entities/         # الكيانات (Account, Transaction, Category)
│   ├── value_objects/    # كائنات القيمة (Money, Name, Note, Color)
│   └── usecases/         # حالات الاستخدام (13+ UseCase)
├── data/
│   ├── local/            # إعدادات Drift و DAOs
│   └── mappers/          # محولات البيانات (Entity <-> Model)
├── application/
│   └── providers/        # الروابط بين الطبقات عبر Riverpod
└── presentation/
    ├── screens/          # الشاشات (Dashboard, Stats, Add, etc.)
    ├── styles/           # AppTheme وأنظمة الألوان المتدرجة
    └── widgets/          # المكونات الزجاجية (Glassmorphism Widgets)
```

## 🚀 دليل المطور (Installation Guide)

1. **تجهيز البيئة:** تأكد من تثبيت Flutter 3.10+.
2. **الاستنساخ:** 
   ```bash
   git clone https://github.com/assem2023-habib/mizaneyati.git
   cd mizaneyati
   ```
3. **الجلب:** `flutter pub get`
4. **توليد الكود البرمجي (خطوة حرجة):**
   ```bash
   flutter pub run build_runner build --delete-conflicting-outputs
   ```
5. **التشغيل:** `flutter run`
