توصيف معماري وتصميمي دقيق لنظام الملاحة (Navigation System)

هذا الملف يشرح بالتفصيل الممل كيف يعمل نظام التنقل في الموقع، الألوان، الحالات، التفاعلات، وكيفية التعامل مع الوضع الليلي والنهاري.

1. بنية نظام الملاحة (The Structure)
يتم تقسيم نظام الملاحة إلى نمطين أساسيين يعتمدان على حجم الشاشة ومكان المستخدم:

أ. الشريط الجانبي (Sidebar Navigation) - للشاشات الكبيرة (Desktop):
- الموقع: ثابت على جهة اليسار (أو اليمين في حالة RTL).
- العرض: عرض ثابت (مثلاً 300px).
- المحتوى: 
  * قسم البروفايل (صورة دائرية + الاسم + روابط التواصل الاجتماعي).
  * قائمة الروابط (Nav Menu).
  * زر تبديل الوضع (Dark/Light Mode Toggle).

ب. الشريط العلوي (Mobile Navigation) - للشاشات الصغيرة (Mobile/Tablets):
- الموقع: ثابت في أعلى الصفحة.
- السلوك: يظهر كزر "Hamburger" عند الضغط عليه يفتح القائمة الجانبية بشكل Overlay يغطي جزءاً من الشاشة.
- التفاعل: يتم إخفاء القائمة تلقائياً عند الضغط على أي رابط أو عند الضغط خارج القائمة.

2. حالات الروابط والتفاعل (Navigation Links States)
لكل رابط في القائمة أربع حالات أساسية مصممة بعناية:

- الحالة العادية (Normal):
  * اللون: رمادي متوسط (مثلاً #a8a9b4) لضمان عدم تشتيت الانتباه.
  * الخلفية: شفافة.
  * الأيقونة: تظهر بجانب النص بلون أخف قليلاً.

- حالة التحويم (Hover):
  * اللون: اللون الأساسي (Primary Color - مثلاً الأزرق أو الأخضر المستخدم).
  * الخلفية: تغيير طفيف جداً في لون الخلفية أو إضافة خط سفلي ناعم.
  * الانتقال: يتم التغيير خلال 0.3 ثانية باستخدام `transition: all 0.3s ease-in-out`.

- الحالة النشطة (Active - ScrollSpy):
  * الاستخدام: عندما يكون المستخدم في قسم معين (مثلاً About)، يتم تمييز رابط About تلقائياً.
  * الشكل: اللون الأساسي للموقع مع زيادة سمك الخط (font-weight: 600) وتغيير لون الأيقونة.

- حالة التركيز (Focus):
  * للأشخاص الذين يستخدمون الكيبورد للتنقل، يظهر إطار ناعم (Outline) حول الرابط لضمان سهولة الوصول (Accessibility).

3. الألوان والسمات (Colors & Themes)
يتم التحكم في الألوان عبر متغيرات CSS (Variables) لضمان سهولة التغيير بين الوضعين:

أ. الوضع النهاري (Light Mode):
- خلفية القائمة: أبيض ناصع (#ffffff).
- لون النص العادي: رمادي غامق (#45505b).
- لون النص النشط: اللون الأساسي (مثلاً #0563af).
- لون الحدود والخطوط الفاصلة: رمادي فاتح جداً (#f2f3f5).

ب. الوضع الليلي (Dark Mode):
- خلفية القائمة: أسود مائل للزرقة غامق جداً (#040b14).
- لون النص العادي: رمادي فاتح (#a8a9b4).
- لون النص النشط: لون ساطع (مثلاً #45a1ff).
- لون الحدود: شفافية خفيفة من الأبيض (rgba(255, 255, 255, 0.1)).

4. منطق الانتقال والتحكم (Navigation Logic)

أ. السكول الناعم (Smooth Scroll):
- عند الضغط على رابط داخلي (مثلاً #about)، لا ينتقل المتصفح فجأة، بل ينزلق بنعومة.
- يتم حساب إزاحة (Offset) للهيدر العلوي في الموبايل لضمان عدم تغطية عنوان القسم.

ب. مراقب السكول (ScrollSpy):
- وظيفة JS تقوم بمراقبة موقع المستخدم الحالي.
- بمجرد دخول "المنطقة المرئية" لقسم معين بنسبة 50%، يتم تحديث الرابط النشط في القائمة فوراً.

ج. زر العودة للأعلى (Back to Top):
- يظهر فقط بعد تجاوز أول 100px من السكول.
- يختفي عند العودة للأعلى.
- الحركة: يظهر بحركة Fade-in وينزلق قليلاً للأعلى.

5. تبديل الوضع (Dark/Light Mode Switch)
- الرمز: أيقونة شمس (للمود النهاري) وأيقونة قمر (للمود الليلي).
- التفاعل: عند الضغط، يتم إضافة Class `.dark-mode` للـ Body.
- الحفظ: يتم استخدام `localStorage` لحفظ خيار المستخدم بحيث يظل الموقع بنفس الوضع عند إعادة التحميل.
- الانتقال: يتم تغيير ألوان الموقع بالكامل بنعومة فائقة باستخدام CSS transitions على خاصية background-color و color.

6. حالات الاستخدام (Use Cases)
- الانتقال بين الصفحات: في صفحات مثل Project Details، يتحول الهيدر الجانبي إلى هيدر علوي ثابت بسيط يحتوي على زر "العودة" واللوغو فقط لتركيز انتباه المستخدم على المحتوى.
- القوائم المنسدلة (Dropdowns): في حال وجود أقسام فرعية (مثلاً خدماتنا -> برمجة، تصميم)، تفتح القائمة بالضغط أو التحويم مع سهم يشير للحالة (مفتوح/مغلق).

7. المواصفات التقنية المقترحة
- المكتبة المفضلة للأيقونات: Bootstrap Icons (خفيفة وسريعة).
- المكتبة المفضلة للأنيميشن: CSS Animations الخام + AOS لظهور العناصر عند السكول.
- منطق الـ JS: Vanilla JS (بدون مكتبات خارجية ثقيلة) لضمان أعلى سرعة تحميل وأداء.
