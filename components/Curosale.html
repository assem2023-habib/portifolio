01التوزيع الهندسي

توزيع الكارتات على الحلقة

الكارتات السبع موزعة بالتساوي على دائرة خيالية باستخدام المثلثات. كل كارت يُحسب موقعه بزاوية ثابتة بينه وبين الكارت التالي.

📐
معادلة توزيع الزوايا
Angle Distribution Formula
// عدد الكارتات N = 7 // الزاوية بين كل كارت والتالي angleStep = 360 / N angleStep = 360 / 7 angleStep = 51.43° ← الزاوية بين كل كارت مجاور // الزاوية النسبية للكارت i بالنسبة للكارت النشط relAngle = ((i - currentIndex) * angleStep + dragDelta) % 360
الكارت الموضع النسبي relAngle الوصف
currentIndex 0° 0° الكارت الأمامي — في الصدارة أمام المشاهد
i + 1 +51.43° 51.43° الكارت التالي — على اليمين قليلاً في الخلف
i + 2 +102.86° 102.86° كارت جانبي أيمن — في العمق
i + 3 +154.28° 154.28° شبه خلفي — يكاد لا يُرى
i + 3.5 ±180° ±180° الكارت الخلفي تماماً — يظهر وجهه الخلفي
i - 2 -102.86° -102.86° كارت جانبي أيسر — في العمق
i - 1 -51.43° -51.43° الكارت السابق — على اليسار قليلاً في الخلف
📏
معادلة الموضع المكاني X و Z
Spatial Position (translateX + translateZ)

بعد حساب الزاوية، نحوّلها إلى إحداثيات فعلية على محوري X وZ باستخدام sin و cos:

// تحويل الزاوية إلى راديان rad = relAngle * (Math.PI / 180) // الإزاحة الأفقية (يمين / يسار) x = Math.sin(rad) * r // الإزاحة العميقة (أمام / خلف) z = Math.cos(rad) * r // r = نصف قطر الحلقة r = 420px ← Desktop r = 260px ← Mobile (≤768px)
الزاويةx = sin(θ) × 420z = cos(θ) × 420الموضع البصري

51.4° +328px +262px يمين — نصف أمام، نصف جانب
90° +420px 0px أقصى اليمين — على مستوى المحور
180° 0px -420px الخلف تماماً — أبعد نقطة

02العمق والمقياس

محاكاة العمق — Scale & Opacity

الكارتات البعيدة تبدو أصغر وأكثر شفافية لمحاكاة المنظور ثلاثي الأبعاد. يُحسب المقياس من موضع Z.

// تطبيع z إلى مدى 0..1 (0=خلف، 1=أمام) scale = (z + r) / (r * 2) // الشفافية: أدنى قيمة 0.30 — أعلى قيمة 1.0 opacity = 0.30 + scale * 0.70 // ضبابية: الكارت النشط صفر، الخلفي أقصاه 2px blur = (1 - scale) * 2 ← px // ترتيب الطبقات (z-index) zIndex = Math.round(scale * 100)
الموضعzscaleopacityblurzIndex
أمام (0°) +420px 1.00 1.00 0px 100
جانب ±51° +262px 0.81 0.87 0.38px 81
جانب ±103° -47px 0.44 0.61 1.12px 44
خلفي (±180°) -420px 0.00 0.30 2px 0
03اتجاه الكارت

rotateY — الكارت يواجه المركز دائماً

كل كارت يدور حول محور Y بمقدار عكسي لزاويته، فيظل واجهاً للمشاهد طوال الدوران. عند الوصول لـ ±180° تظهر الواجهة الخلفية تلقائياً.

// تطبيع relAngle إلى النطاق [-180, +180] let normAngle = relAngle % 360 if (normAngle > 180) normAngle -= 360 if (normAngle < -180) normAngle += 360 // الدوران المضاد (الكارت يواجه المركز) rotY = -normAngle
موضع الكارتrelAnglerotY المُطبَّقما يراه المشاهد
🟢 أمام 0° rotateY(0°) الواجهة الأمامية بالكامل
➡ يمين +51.4° rotateY(-51.4°) الواجهة الأمامية مائلة، واجهة لليمين
⬆ جانب أيمن +90° rotateY(-90°) حافة الكارت فقط (edge)
🔴 خلف ±180° rotateY(∓180°) الواجهة الخلفية (card-back)
⬆ جانب أيسر -90° rotateY(+90°) حافة الكارت، الواجهة تتجه لليسار
⬅ يسار -51.4° rotateY(+51.4°) الواجهة الأمامية مائلة، واجهة لليسار
💡
سر الـ Backface: كلٌّ من .card-inner (الأمامية) و.card-back (الخلفية) لديهما backface-visibility: hidden. والخلفية مُدوَّرة مسبقاً بـ rotateY(180deg). لذا حين يصل الكارت لـ ±180°، الأمامية تختفي والخلفية تظهر — تلقائياً بدون أي JS.
04نظام الإمالة

rotateZ — إمالة الكارتات عند الانتقال

عند الضغط على Next/Prev أو السحب، تميل الكارتات على محور Z لإعطاء إحساس الحركة والزخم. يتحرك الميل بـ Exponential Easing حتى يعود للصفر.

⚙️
ثوابت النظام
Tilt Constants
الثابتالقيمة
TILT_MAX 14°
TILT_EASE 0.10
tiltTarget reset 180ms
stop threshold 0.05°
🎯
الاتجاه والإشارة
Direction Sign
الحدثdirtiltTarget
Next ← -1 -14°
Prev → +1 +14°
Auto Play -1 -14°
بعد 180ms — 0°
📈
معادلة Easing التدريجي
Exponential Easing per Frame
// كل إطار في requestAnimationFrame: tiltAngle += (tiltTarget - tiltAngle) * 0.10 // مثال: من 0° نحو -14° (next pressed) // Frame 1: tiltAngle = 0 + (-14 - 0) * 0.10 = -1.4° // Frame 2: tiltAngle = -1.4 + (-14-(-1.4)) * 0.10 = -2.66° // Frame 10: tiltAngle ≈ -8.9° // Frame 20: tiltAngle ≈ -12.1° // ثم بعد 180ms: tiltTarget = 0 → يعود بنفس الطريقة // إيقاف الحلقة عند الاقتراب الكافي if (Math.abs(tiltAngle - tiltTarget) < 0.05) stopAnimation()
🃏
إمالة كل كارت بحسب عمقه
Per-Card Tilt Weight

الكارت النشط (scale=1) يأخذ الإمالة الكاملة، الكارتات الخلفية أقل — يُعطي إحساس العمق الحقيقي:

// وزن الإمالة = 50% ثابت + 50% حسب العمق cardTilt = activeTilt * (0.5 + scale * 0.5) // مثال عند activeTilt = -14°: // الكارت الأمامي (scale=1.00): cardTilt = -14 * 1.0 = -14.0° // الكارت ±51° (scale=0.81): cardTilt = -14 * 0.91 = -12.7° // الكارت ±103° (scale=0.44): cardTilt = -14 * 0.72 = -10.1° // الكارت الخلفي (scale=0.00): cardTilt = -14 * 0.5 = -7.0°
⏱
التسلسل الزمني للإمالة عند Autoplay (2800ms interval):
t=0ms — triggerTilt(-1) → tiltTarget = -14° → تبدأ حلقة rAF
t=180ms — tiltTarget = 0° → بدأ العودة
t≈600ms — tiltAngle ≈ 0° (تقريباً) → توقف rAF
t=2800ms — next() يُطلق من جديد
05نظام الانحراف

skewX — انحراف الكارت أثناء الحركة

يُضاف skewX بنسبة مباشرة من قيمة tiltAngle. يُعطي إحساساً بالسرعة والدفع أثناء الانتقال.

// skewX مشتق مباشرة من activeTilt skewVal = activeTilt * 0.38 // مثال عند ذروة الإمالة (activeTilt = -14°): skewVal = -14 * 0.38 = -5.32° ← ذروة الانحراف // عند السحب يدوياً: activeTilt = -dragDelta * 0.45 skewVal = (-dragDelta * 0.45) * 0.38 skewVal = -dragDelta * 0.171 ← معامل السحب
الحالةactiveTiltskewVal
راحة (لا حركة) 0° 0°
ذروة Next -14° -5.32°
ذروة Prev +14° +5.32°
سحب 50px يساراً -2.25°-0.86°
سحب 200px يساراً -9° -3.42°
06مكدس التحويلات

التحويل الكامل — ترتيب العمليات

الترتيب في CSS transform مهم جداً — كل تحويل يؤثر على إحداثيات التحويل التالي. هذا هو الترتيب المُطبَّق على كل كارت:

1
translateX(x)
sin(relAngle) × r
ينقل الكارت أفقياً على محور X
2
translateZ(z)
cos(relAngle) × r
ينقله للأمام/خلف — يعطي العمق البصري
3
rotateY(rotY)
-normAngle
يوجه واجهة الكارت نحو المشاهد دائماً
4
rotateZ(cardTilt)
activeTilt × (0.5 + scale×0.5)
يميل الكارت بزاوية الحركة (tilt effect)
5
skewX(skewVal)
activeTilt × 0.38
ينحرف الكارت — يُعبّر عن السرعة
// السطر الكامل في JS card.style.transform = ` translateX(${x}px) translateZ(${z}px) rotateY(${rotY}deg) rotateZ(${cardTilt}deg) skewX(${skewVal}deg) `
⚠️
ملاحظة ترتيب العمليات: translateX/Z أولاً لأنهما يضعان الكارت في موقعه على الحلقة. ثم rotateY ليوجهه. ثم rotateZ وskewX اللذان يُطبَّقان على الكارت المُوجَّه — لو أُخِّر rotateY لتغير محور الإمالة وانكسر التأثير.

08تفاعل السحب

Drag & Swipe — التفاعل اليدوي

يدعم النظام السحب بالماوس، اللمس، ولوحة المفاتيح. المعامل الرئيسي يُحوِّل الـ pixels لدرجات دوران.

// تحويل المسافة إلى دوران (أثناء السحب) dragDelta = (clientX - dragStartX) * 0.10 // مثال: سحب 100px يميناً dragDelta = 100 * 0.10 = 10° ← دوران 10 درجات // عتبة القرار (commit to next/prev) if (Math.abs(moved) > 60px) ← يُنفِّذ الانتقال else ← يعود للموضع الأصلي // الإمالة أثناء السحب (لا rAF — مباشر) activeTilt = -dragDelta * 0.45
الحدثالمعاملالقيمةالتأثير
mousemove / touchmove dragSensitivity 0.10 10px = 1° دوران
تأثير الإمالة أثناء السحب tiltFromDrag 0.45 dragDelta × -0.45 → tilt
عتبة تأكيد الانتقال threshold 60px أقل من 60px → رجوع للأصل
لوحة المفاتيح ← ArrowLeft prev() يُطلق تأثير الإمالة الكامل
لوحة المفاتيح → ArrowRight next() يُطلق تأثير الإمالة الكامل
