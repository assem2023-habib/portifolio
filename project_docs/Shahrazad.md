# مشروع شهرزاد (Shehrezad Backend)

![Logo](logo.jpg)

## نظرة عامة

هذا المشروع هو النظام الخلفي (Backend) لمنصة **شهرزاد**، وهي منصة متكاملة لإدارة التجارة الإلكترونية وعلاقات العملاء. يوفر النظام واجهات برمجة تطبيقات (APIs) قوية لإدارة المنتجات، الطلبات، المخزون، الديون، والتقارير المالية، مع دعم كامل لتطبيقات العملاء ولوحة تحكم الإدارة.

## ماذا يقدّم هذا المشروع؟

- متجر إلكتروني متكامل لإدارة المنتجات، الصور، المخزون، والأسعار.
- نظام طلبات كامل من السلة وحتى إنشاء الفاتورة وتأكيد الاستلام.
- نظام ديون وحسابات للعملاء مع ملخصات وتقارير تاريخية.
- طبقة مصادقة وصلاحيات (RBAC) للعميل والموظف وـ Super Admin.
- لوحة تحكم غنية لإدارة العملاء، الطلبات، الديون، الكوبونات، والإعدادات.
- نظام إشعارات مدمج عبر Firebase لإرسال التنبيهات للعملاء والموظفين.
- مهام مجدولة (Cron Jobs) لمعالجة السلال المتروكة وتحديث حالات الطلبات.

---

## التقنيات المستخدمة

يعتمد النظام على بنية تقنية حديثة ومستقرة:

- **Node.js & Express**: كإطار عمل أساسي للسيرفر.
- **MySQL**: لقاعدة البيانات العلائقية.
- **Firebase Admin SDK**: لإدارة الـ Push Notifications وتخزين الملفات (Firebase Storage).
- **Cloudinary**: (سابقاً) لإدارة الصور.
- **JWT (JsonWebToken)**: لتأمين الواجهات والتحقق من الهوية.
- **Node-cron**: لجدولة المهام المؤتمتة (مثل تنظيف السلال المهجورة).
- **Puppeteer & PDFKit**: لتوليد الفواتير والتقارير بصيغة PDF.
- **Multer**: لمعالجة رفع الملفات والصور.

## هيكلية المشروع (Project Structure)

```text
src/
├── config_new/       # إعدادات قاعدة البيانات، فايربيز، والمهام المجدولة
├── middleware_new/   # برمجيات الوسيطة (التحقق من التوكن، الصلاحيات، التحقق من البيانات)
├── models_new/       # تعريفات جداول قاعدة البيانات (منطق البيانات)
├── modules_new/      # الوحدات الوظيفية للنظام (النواة)
│   ├── auth/         # المصادقة (للعملاء، الموظفين، ولوحة التحكم)
│   ├── common/       # ميزات مشتركة (مثل المواقع/المناطق)
│   ├── customer/     # كافة العمليات الخاصة بتطبيق العميل
│   ├── dashboard/    # كافة العمليات الخاصة بلوحة تحكم المسؤولين
│   └── notifications/# نظام الإشعارات
├── app.js            # إعداد مسارات Express والبرمجيات الوسيطة
server.js             # نقطة الانطلاق الأساسية للسيرفر
```

## الوحدات الرئيسية في النظام

### 1. المصادقة والصلاحيات (Auth)

المجلد: `src/modules_new/auth`

- تسجيل دخول العميل (Customer) ولوحة التحكم (Dashboard).
- تسجيل دخول Super Admin وإنشاء أول Super Admin.
- إدارة جلسات الدخول والخروج (Login / Logout).
- جلب بيانات المستخدم الحالي `/auth/me`.
- تحديث وحفظ FCM Token للمستخدم.

### 2. واجهات تطبيق العميل (Customer Modules)

المجلد: `src/modules_new/customer`

- **products**  
  استعراض المنتجات، الفلترة حسب التصنيف، جلب تفاصيل منتج، عرض الكوبونات والأسعار بعد الخصم.

- **cart**  
  إدارة سلة المشتريات: إضافة/حذف منتجات، تعديل الكميات، قفل العناصر، حساب المجموع وربط السلة بالطلب النهائي.

- **orders**  
  جلب طلبات العميل مع أو بدون Pagination، جلب تفاصيل طلب واحد، تأكيد استلام الطلب، وحساب إحصائيات الطلبات حسب الحالة.

- **debts**  
  عرض ملخص ديون العميل، سجل الحركات، وتفاصيل كل دين مرتبط بالطلبات.

- **coupons**  
  التحقق من كود الخصم، ربطه بالسلة أو المنتجات، وحساب الخصم النهائي.

- **favorites**  
  إدارة قائمة المفضلة للعميل (إضافة/حذف/جلب العناصر المفضلة).

- **reviews**  
  إضافة تقييمات للمنتجات، جلب تقييمات المستخدم، وإدارة حالة التقييم (معلّق، مقبول، مرفوض).

- **profile**  
  تعديل بيانات الحساب، رقم الهاتف، البريد الإلكتروني، الصورة، والعنوان.

- **settings**  
  جلب إعدادات عامة يحتاجها تطبيق العميل (مثل أرقام الدعم، سياسات الشحن…).

- **support**  
  جلب أرقام دعم الموظفين المخصصين لخدمة العملاء.

- **invoices**  
  جلب فواتير المشتريات الخاصة بالعميل واستعراضها.

### 3. واجهات لوحة التحكم (Dashboard Modules)

المجلد: `src/modules_new/dashboard`

- **admins**  
  إدارة موظفي النظام (إضافة/تعديل/تعطيل)، وتحديد الأدوار: Admin, Accountant, Super Admin.

- **customers**  
  جلب العملاء، البحث والفلترة، مراجعة تفاصيل العميل وحالة الديون والطلبات.

- **product**  
  إضافة منتج جديد، رفع صور متعددة، إدارة الألوان والمقاسات والمخزون، تحديث المنتج أو حذفه.

- **carts**  
  استعراض السلال من طرف الإدارة، تحويل السلة إلى طلب، أو مراجعة سلال متأخرة/متروكة.

- **orders**  
  جلب الطلبات مع فلاتر متعددة (حالة، مكان، بحث)، إحصائيات الطلبات، تحديث حالة الطلب، تعديل الإجمالي، حذف الطلب، وتوليد الفواتير PDF.

- **debts**  
  إدارة ديون العملاء: إنشاء دين، تعديل حالة الدفع، عرض ملخص الديون والتقارير.

- **coupons**  
  إنشاء كوبونات جديدة، تعديلها، تفعيل/تعطيل، وإدارة شروط الخصم وحدّ الخصم الأقصى.

- **reviews**  
  مراجعة تقييمات العملاء للمنتجات، قبولها أو رفضها، وعرض إحصائيات التقييمات.

- **images**  
  مكتبة صور مشتركة لرفع وحذف الملفات والصور المرتبطة بالمنتجات أو المحتوى.

- **settings**  
  ضبط الإعدادات العامة للنظام:  
  رقم الشحن، أيام الشحن، معلومات الحوالات البنكية/المكتبية، أرقام الدعم والاستفسار، حدود السلة والمنتجات.

- **statistics**  
  إحصائيات عامة للمبيعات، الطلبات، سلوك العملاء، وأكثر المنتجات مبيعاً (تُستهلك من لوحة التحكم).

### 4. نظام الإشعارات (Notifications)

المجلد: `src/modules_new/notifications`

- التكامل مع Firebase Admin لإرسال إشعارات Push.
- حفظ الإشعارات في قاعدة البيانات مع حالة القراءة.
- دعم الإرسال لفئة معينة من المستخدمين أو لمستخدم محدد.

### 5. الإعدادات وقاعدة البيانات

- `src/config_new`  
  الاتصال بقاعدة البيانات، إعداد الجداول، ملفات الـ migrations، إعدادات Cron، وملف الثوابت العامة (constants).
- `src/models_new`  
  تعريف كائنات البيانات (Models) المطابقة للجداول: Users, Products, Orders, OrderItems, Cart, Debts, Reviews, Notifications, وغيرها.

## واجهات النظام الوظيفية (API Reference)

تم تقسيم النظام إلى منطقتين رئيسيتين:

### 1. واجهات تطبيق العميل (Customer API)

تتمركز هذه الواجهات تحت المسار `/v2/customer` وتخدم احتياجات المتسوقين:

- **المصادقة (Auth)**: التسجيل، تسجيل الدخول، واستعادة كلمة المرور.
- **المنتجات (Products)**: استعراض المنتجات، البحث حسب التصنيف، وتفاصيل المنتج.
- **السلة (Cart)**: إضافة المنتجات، تحديث الكميات، وحجز السلة.
- **المفضلة (Favorites)**: إدارة قائمة الأمنيات الخاصة بالعميل.
- **الطلبات (Orders)**: إنشاء طلبات جديدة، تتبع حالة الطلب، وتاريخ الطلبات السابقة.
- **الديون (Debts)**: متابعة المبالغ المستحقة على العميل وكشوف الحساب.
- **الكوبونات (Coupons)**: التحقق من كود الخصم وتطبيقه.
- **الملف الشخصي (Profile)**: تحديث بيانات العميل، العناوين، وإدارة التوكنات (FCM).
- **الدعم الفني (Support)**: إرسال تذاكر الدعم والتواصل.
- **الفواتير (Invoices)**: استعراض وتحميل فواتير المشتريات.

### 2. واجهات لوحة التحكم (Dashboard API)

تتمركز تحت المسار `/v2/dashboard` وتدير كافة جوانب النظام الإداري:

- **الإدارة (Admins)**: إدارة حسابات الموظفين وصلاحياتهم (Super Admin, Admin, Accountant).
- **المنتجات (Product Management)**: إضافة، تعديل، حذف، وتحديث مخزون المنتجات مع رفع الصور.
- **العملاء (Customers)**: إدارة بيانات العملاء، حظر الحسابات، ومراجعة سجلاتهم.
- **الطلبات (Order Management)**: معالجة الطلبات، تغيير حالاتها (قيد التحضير، تم الشحن، الخ)، وإصدار الفواتير.
- **الصور (Images)**: إدارة مكتبة الصور المرفوعة على السيرفر أو Firebase.
- **الديون والمحاسبة (Debts & Accounting)**: تسجيل الدفعات، متابعة ديون العملاء، والعمليات المالية.
- **الإحصائيات (Statistics)**: تقارير المبيعات، أكثر المنتجات مبيعاً، وإحصائيات المستخدمين.
- **الإعدادات (Settings)**: التحكم في إعدادات المتجر العامة (أسعار الشحن، العناوين، معلومات التواصل).
- **الإشعارات (Notifications)**: إرسال إشعارات جماعية أو فردية للعملاء.

## التثبيت والتشغيل

1. **تثبيت المكتبات**:

   ```bash
   npm install
   ```

2. **إعداد ملف البيئة**:
   قم بإنشاء ملف `.env` في المجلد الرئيسي وأضف المتغيرات التالية:

   ```env
   PORT=4000
   DB_HOST=localhost
   DB_USER=root
   DB_PASS=your_password
   DB_NAME=shehrezad_db
   JWT_SECRET=your_secret_key
   # إعدادات Firebase
   FIREBASE_PROJECT_ID=...
   ```

3. **تشغيل السيرفر**:
   ```bash
   npm start
   ```
   أو في بيئة التطوير باستخدام nodemon:
   ```bash
   npm run dev
   ```

## المهام المجدولة (Cron Jobs)

يحتوي النظام على مهام تعمل تلقائياً في الخلفية:

- **تنظيف السلال**: حذف المنتجات غير المحجوزة من السلال بعد فترة زمنية محددة.
- **تحديث الحالات**: مراجعة تلقائية لحالات الطلبات أو العروض المنتهية.

## ملاحظات إضافية

- يدعم النظام نظام صلاحيات دقيق (RBAC) يضمن وصول كل مستخدم للبيانات المسموح له بها فقط.
- يتم تسجيل العمليات الحساسة لضمان أمان النظام.
- النظام مجهز للعمل خلف بروكسي (مثل Nginx) مع دعم `trust proxy`.
