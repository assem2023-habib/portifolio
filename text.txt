وصف معماري تفصيلي لموقع البورتفوليو (نسخة Static جاهزة للتطوير مستقبلاً)

------------------------------------
1) نظرة عامة على المشروع
------------------------------------

- نوع الموقع:
  - موقع بورتفوليو شخصي لمطوّر برمجيات، يُعرّف بالمهارات، الخدمات، الأعمال السابقة، وآراء العملاء.
  - يظل Static بالكامل: صفحات HTML + CSS + JavaScript بدون Backend ديناميكي.
- الهدف المعماري:
  - بناء هيكل واضح وقابل للتطوير لاحقاً (إلى React أو أي Framework) مع الحفاظ على نفس تقسيم الشاشات والأقسام.
  - الفصل بين:
    - المحتوى (البيانات: مشاريع، خدمات، شهادات).
    - العرض (HTML/CSS).
    - السلوك (JavaScript مقسّم حسب الشاشة/الوحدة).

------------------------------------
2) هيكل الصفحات (الشاشات الرئيسية)
------------------------------------

النسخة الجديدة للموقع تتكوّن من الصفحات التالية:

1. صفحة Home (الرئيسية)
   - مسار: index.html
   - تحتوي على: Header/Sidebar Navigation, Hero, About, Skills, Services (مختصرة), Featured Portfolio, Testimonials, Contact, Footer.

2. صفحة All Projects (كل المشاريع)
   - مسار: projects.html
   - شبكة بكل المشاريع مع فلاتر حسب النوع (Web / Mobile / Desktop) + إمكانية البحث.

3. صفحة Project Details (تفاصيل مشروع)
   - مسار: project.html?project={id}
   - تفاصيل مشروع واحد + معرض صور + قائمة Projects أخرى للانتقال السريع.

4. صفحة Services (كل الخدمات)
   - مسار: services.html
   - عرض لكل الخدمات بتقسيم بصري واضح (أيقونات + وصف + قائمة Features مختصرة).

5. صفحة Service Details (تفاصيل خدمة)
   - مسار: service-details.html?service={id}
   - تفاصيل خدمة واحدة + Sidebar يحتوي قائمة بكل الخدمات.

6. صفحة Contact (اختيارية مستقلة)
   - مسار: contact.html (يمكن الاكتفاء بقسم في Home، لكن يُفضَّل ملف منفصل مستقبلاً).

7. صفحة About مستقلة (اختيارية)
   - مسار: about.html
   - سيرة ذاتية موسّعة، Timeline، وقصّة شخصية أكثر تفصيلاً.

------------------------------------
3) التصميم العام (Layout, Grid, Typo, Colors)
------------------------------------

3.1) الشبكة والتوزيع (Layout/Grid)

- استخدام CSS Grid + Flexbox كأساس:
  - الهيكل الأساسي لصفحات المحتوى: حاوية بعرض ثابت max-width (مثلاً 1140px–1200px) في المنتصف.
  - تقسيم الصفحات المعقّدة (مثل Home) إلى أقسام section متتالية بارتفاع رأسي مريح (60–100px padding عمودي).
- Breakpoints مقترحة:
  - Mobile: حتى 575px
  - Tablet: 576–991px
  - Small Desktop: 992–1199px
  - Large Desktop: 1200px فأعلى
- سلوك عام:
  - في الشاشات الكبيرة: Header جانبي ثابت (sidebar) مع قائمة عمودية.
  - في الشاشات الصغيرة: Header علوي قابل للفتح/الإغلاق (Offcanvas) مع زرّ Hamburger.

مكتبات مساعدة (اختيارية):
  - يمكن الاكتفاء بـ CSS خالص + Flexbox/Grid بدون Framework.
  - أو استخدام Bootstrap 5 كـ Grid جاهز:
    - ميزة: سرعة التطوير.
    - عيب: زيادة حجم CSS وقيود في التخصيص.
  - بديل أخف: PicoCSS أو Skeleton، لكن يفضّل الاستمرار مع Bootstrap 5 هنا لأنه مستخدم أساساً في النسخة القديمة.

3.2) الألوان (Color System)

تعريف نظام ألوان باستخدام CSS Variables في :root:

- ألوان أساسية:
  - background-main: #0b0f19 (خلفية داكنة مريحة).
  - background-section: #0f172a (لأقسام مميّزة مثل Portfolio / Services).
  - surface: #111827 (بطاقات، Cards).
  - primary: #3b82f6 (أزرق مائل قليلاً، مناسب للأزرار والروابط).
  - accent: #22c55e (أخضر خفيف لعناصر ثانوية أو highlights).
  - text-main: #e5e7eb (نص أساسي مفتوح).
  - text-muted: #9ca3af (نص ثانوي).
  - border-subtle: rgba(148, 163, 184, 0.2).

- استخدام الألوان:
  - Hero: تدرّج غامق: background-main مع Overlay بسيط.
  - Sections البديلة (مثل Services, Portfolio, Testimonials): background-section.
  - Cards (الخدمات، المشاريع): surface + border-subtle.
  - Primary buttons & links: primary.
  - Hover states: تفتيح 10–15% من اللون الأساسي (يمكن استخدام color-mix أو درجات ثابتة).

3.3) الخطوط (Typography)

- لواجهة إنجليزية:
  - Heading font: "Poppins", system-ui, -apple-system, BlinkMacSystemFont, "Segoe UI", sans-serif.
  - Body font: "Inter", system-ui, sans-serif.
- في حالة دعم العربية لاحقاً:
  - Arabic heading/body: "Cairo", "Tajawal", "Noto Sans Arabic", sans-serif.

- مستويات النص:
  - H1 (مثلاً اسمك في Hero): 48–64px Desktop, 32–40px Mobile.
  - H2 (عناوين الأقسام): 28–32px Desktop, 22–26px Mobile.
  - H3/H4 للعناوين الداخلية: 18–22px.
  - Body: 14–16px مع line-height 1.6–1.8.

3.4) الأيقونات

- المكتبة الحالية: Bootstrap Icons (مقبولة جداً وخفيفة).
- بدائل محتملة:
  - Heroicons (SVG) – جميل لكن يتطلب إدارة SVG.
  - Font Awesome – شهير لكن أثقل من Bootstrap Icons.
- التوصية:
  - الاستمرار مع Bootstrap Icons للموقع Static، لأنها متوافقة أصلاً مع التصميم القديم وخفيفة.

------------------------------------
4) السلوك العام (Navigations, Scroll, Animations)
------------------------------------

4.1) الهيدر والتنقل (Header & Navigation)

- في Home:
  - Header ثابت على يسار الشاشة (Desktop) يحتوي:
    - صورة شخصية صغيرة.
    - اسم + عنوان وظيفي.
    - قائمة روابط تنقلك إلى أقسام الصفحة (About, Skills, Services, Portfolio, Testimonials, Contact).
  - عند الضغط على أي رابط:
    - يتم Scroll سلس إلى القسم المقابل.
    - لا يعاد تحميل الصفحة (Anchor links فقط).

- في الصفحات الفرعية (Projects, Project Details, Services, Service Details):
  - Header علوي بسيط (Top Nav) يحتوي:
    - Logo/Name.
    - Links: Home, Projects, Services, Contact.
    - زرّ العودة للأعلى (اختياري في النوافذ الطويلة).

- السلوك في الموبايل:
  - زر Hamburger في أعلى يمين/يسار.
  - عند الضغط:
    - تظهر قائمة كاملة الشاشة (Overlay) أو Offcanvas من الجانب.
  - عند اختيار رابط:
    - تغلق القائمة.

مكتبة مقترحة:
  - يمكن تنفيذ كل ذلك بـ JavaScript بسيط (toggle class).
  - لا حاجة لمكتبة خاصة للـ Nav.

4.2) سلوك الـ Scroll

- Smooth Scroll:
  - استخدام CSS: scroll-behavior: smooth على html.
  - أو JavaScript بسيط للمتصفحات الأقدم (اختياري).

- Scrollspy:
  - كود JS بسيط:
    - يستمع لحدث scroll.
    - يحدد section الظاهر حالياً بناء على window.scrollY + offset.
    - يضيف class active على الرابط المناسب في nav.
  - بديل: استخدام IntersectionObserver لمراقبة كل section وتحديث الـ Nav بناءً على العنصر المتقاطع مع منتصف الشاشة.

- زرّ Scroll to Top:
  - زر صغير دائري في أسفل يمين/يسار.
  - يظهر فقط بعد scroll لمسافة معينة (مثلاً 400px).
  - عند الضغط: scroll إلى أعلى الصفحة بسلاسة.

4.3) الأنيميشن عند الظهور (On-Scroll Animations)

- المكتبة الحالية: AOS (Animate On Scroll).
  - سهلة الاستخدام (data-aos="fade-up" إلخ).
  - مناسبة جداً لموقع static.
- بدائل:
  - ScrollReveal.js – مرنة أكثر لكن تحتاج JS configuration.
  - GSAP + ScrollTrigger – قوية جداً لكن زائدة عن الحاجة لبورتفوليو بسيط.
- التوصية:
  - الاستمرار مع AOS في النسخة الجديدة.
  - تنظيم استخدام data-aos بحيث:
    - Hero يظهر بـ zoom/fade.
    - البطاقات (Cards) تظهر بـ fade-up staggered.
    - الشهادات تظهر بـ fade-left/right بسيط.

4.4) الـ Preloader

- شاشة بسيطة:
  - خلفية بـ background-main.
  - دائرة تحميل أو Logo.
- السلوك:
  - تظهر حتى انتهاء تحميل الصور الرئيسية أو حتى أول paint.
  - بعد load أو setTimeout (1–1.5s)، يتم إضافة class fade-out ثم إزالة العنصر من DOM.

4.5) Dark/Light Mode (اختياري)

- بالتفصيل:
  - body يحمل class theme-light أو theme-dark.
  - تعريف متغيرات الألوان لكل Theme في CSS.
  - زر صغير في الـ Header يبدّل بين الثيمين.
  - حفظ الاختيار في localStorage.

------------------------------------
5) صفحة Home (index.html)
------------------------------------

5.1) Header / Sidebar

- المحتوى:
  - صورة شخصية صغيرة (دائرية).
  - اسم المطوّر + Title (مثلاً: Full Stack Developer).
  - قائمة روابط:
    - About
    - Skills
    - Services
    - Portfolio
    - Testimonials
    - Contact
  - روابط Social (GitHub, LinkedIn, Email).

- السلوك:
  - في Desktop: Sidebar ثابت مع ارتفاع كامل.
  - في Mobile: Offcanvas يظهر/يختفي بالزر.
  - تمييز الرابط النشط باستخدام Scrollspy.

5.2) Hero Section

- الهدف:
  - تقديم اسمك، دورك، وصف مختصر، وروابط رئيسية.

- المحتوى:
  - خلفية:
    - صورة يمكنك الحفاظ عليها (صورة احترافية) + Overlay بتدرّج غامق.
  - نص:
    - H1 باسمك.
    - سطر وصف مع تأثير typed:
      - "Full Stack Web Developer"
      - "Laravel & React Specialist"
      - "Backend Engineer"
  - أزرار:
    - زر رئيسي: "View Portfolio" → scroll إلى قسم Portfolio.
    - زر ثانوي: "Download CV" → تحميل ملف PDF.

- المكتبات:
  - Typed.js لتأثير الكتابة (كما هو مستخدم الآن).
  - بديل أخف:
    - تنفيذ بسيط بـ JavaScript (تبديل نصوص كل 3–4 ثوانٍ بدون typing effect).

5.3) About Section

- المحتوى:
  - صورة شخصية كبيرة أو نفس صورة الـ profile.
  - نص تعريفي (من 2–3 فقرات).
  - قائمة معلومات شخصية (العمر، البلد، البريد، LinkedIn, Freelance status).

- السلوك:
  - تقسيم إلى عمودين في Desktop (صورة + نص).
  - عمود واحد فوق الآخر في Mobile.
  - استخدام AOS: الصورة بـ fade-right، النص بـ fade-left.

5.4) Skills Section

- الهدف:
  - إبراز المهارات التقنية والأدوات.

- المحتوى:
  - تقسيم إلى مجموعات:
    - Backend (Laravel, PHP, Node…).
    - Frontend (React, Vue, JS, CSS).
    - DevOps/Tools (Git, Docker, CI/CD).
  - لكل مهارة:
    - اسم المهارة.
    - مستوى (مثلاً شريط progress أو Badge: Intermediate, Advanced).

- المكتبات:
  - يمكن استخدام PureCounter فقط للعدادات الكلّية (عدد المشاريع، عدد العملاء، سنوات الخبرة).
  - يفضّل عدم المبالغة في progress bars؛ يمكن استخدام CSS بسيط.

5.5) Services (Highlights)

- يعرض 3–6 خدمات رئيسية.
- لكل خدمة:
  - Icon (Bootstrap Icon يعبّر عن الخدمة).
  - Title.
  - Short description.
  - Link "View Details" → service-details.html?service={id}.

- التوزيع:
  - Grid من 3 أعمدة في Desktop، 2 في Tablet، 1 في Mobile.

- الأنيميشن:
  - ظهور متدرج من الأسفل مع delay متزايد لكل بطاقة (AOS).

5.6) Featured Portfolio

- الهدف:
  - عرض 6 مشاريع مختارة (من 9 المتاحة في البيانات).

- المحتوى:
  - Grid (مستوحى من masonry) مع استخدام CSS Grid + ارتفاعات متقاربة.
  - لكل مشروع:
    - صورة preview.
    - عنوان.
    - تصنيف (Badge: Web/Mobile/Desktop).
    - اختيارياً Tag للتقنيات الأساسية (Laravel, React…).
    - أيقونتان:
      - Preview: تفتح الصورة في Lightbox.
      - Details: تذهب إلى project.html?project={id}.

- المكتبات:
  - Lightbox:
    - GLightbox (ممتاز وموجود بالفعل).
    - بديل: Photoswipe (أثقل قليلاً).
  - فلترة:
    - في Home يمكن الاكتفاء بعرض Featured فقط بدون فلاتر.

5.7) Testimonials

- المحتوى:
  - Slider يحتوي عناصر:
    - صورة الشخص.
    - الاسم + الدور.
    - نص الشهادة.
    - تقييم نجوم (1–5).

- المكتبة:
  - Swiper.js:
    - Slider واحد في Home، إعدادات:
      - slidesPerView: 1 (Mobile), 2 (Desktop).
      - autoplay: true.
      - loop: true.
  - بديل:
    - Tiny-slider أو Flickity، لكن Swiper ما زال خياراً ممتازاً.

5.8) Contact Section

- المحتوى:
  - عمود:
    - معلومات التواصل (Email, LinkedIn, GitHub).
    - قائمة سريعة بالـ Services.
  - عمود:
    - Form:
      - الاسم، البريد، الموضوع، الرسالة.
      - زر "Send Message".

- السلوك:
  - في النسخة Static:
    - استخدام خدمة خارجية (مثل Formspree أو EmailJS) لمعالجة الرسائل.
    - أو إرسال البريد مباشرة عبر mailto (أبسط لكن أقل احترافية).

6) صفحة All Projects (projects.html)
------------------------------------

6.1) التخطيط

- Page Title أعلى الصفحة:
  - Breadcrumb: Home > Portfolio > All Projects.
  - H1: All Projects.
  - وصف قصير.

- قسم الفلاتر:
  - أزرار:
    - All
    - Web
    - Mobile
    - Desktop
  - اختيارياً حقل بحث نصي (حسب عنوان المشروع).

- Grid المشاريع:
  - استخدام نفس card structure كما في Home لكن لكل المشاريع التسعة.
  - توزيع:
    - 3 أعمدة Desktop.
    - 2 أعمدة Tablet.
    - 1 عمود Mobile.

6.2) السلوك

- الفلاتر:
  - يمكن إبقاء Isotope (مستخدم حالياً) إن أردت:
    - ميزة: فلترة/ترتيب سلس مع أنيميشن جاهز.
    - عيب: مكتبة إضافية بحجم لا بأس به.
  - بديل أبسط:
    - فلترة بـ JavaScript:
      - عند الضغط على فلتر: إخفاء/إظهار العناصر حسب data-category.
      - يمكن إضافة transition على opacity/transform.

- البحث:
  - Event on input:
    - يفلتر العناصر حسب match في عنوان المشروع أو التقنيات.

- Lightbox:
  - كما في Home (GLightbox).

7) صفحة Project Details (project.html)
------------------------------------

7.1) التخطيط

- Page Title مع Breadcrumb:
  - Home > Portfolio > {Project Title}.
- قسم المحتوى الرئيسي:
  - يسار:
    - Slider (Swiper) لصور المشروع.
  - يمين:
    - Card بمعلومات المشروع:
      - Category.
      - Client (اختياري).
      - Technologies.
      - Status.
      - Project URL (زر).
  - أسفل:
    - وصف مطوّل.
    - قائمة Features (bullet points).
    - اختيارياً Testimonial خاص بالمشروع (إن وجد).

- Sidebar (اختياري يمين/يسار أو أسفل):
  - قائمة "More Projects":
    - روابط لـ 4–6 مشاريع أخرى.

7.2) السلوك

- قراءة id من query string (project في URL).
- جلب البيانات من ملف بيانات JavaScript (portfolioData).
- تعبئة:
  - العنوان، الوصف، الحقول الجانبية، قائمة features، الصور.
- Swiper:
  - إعدادات بسيطة: slidesPerView: 1، Arrows + Pagination.

------------------------------------
10) البيانات والمكتبات المقترحة
------------------------------------

10.1) طبقة البيانات (Data Layer)

- ملف مستقل لكل نوع:
  - data/services.js
  - data/portfolio.js
  - data/testimonials.js
- كل ملف يصدّر كائن/مصفوفة:
  - services: id, title, description, image, features[], details.
  - portfolio: id, title, category, image, description, technologies, status, url, features[], galleryImages[].
  - testimonials: id, name, role, image, content, rating.

10.2) المكتبات الحالية مقابل البدائل

- Bootstrap:
  - الإبقاء عليه مبدئياً لأنه يقلل وقت بناء الـ layout.
  - بديله: CSS خالص + Utility classes مخصّصة.

- Bootstrap Icons:
  - الإبقاء عليها (خفيفة ومتجانسة).

- AOS:
  - الإبقاء عليها لأنها بسيطة وتؤدي الغرض.

- Swiper:
  - الإبقاء عليه للسلايدر (Hero/Testimonials/Project Gallery).

- GLightbox:
  - الإبقاء عليه لـ portfolio preview.

- Isotope:
  - يفضّل استبداله بفلترة JS بسيطة من أجل خفة المشروع، إلا إذا كنت تحتاج أنيميشن متقدمة في Grid.

- Typed.js:
  - يمكن الإبقاء عليه في Hero أو استبداله بـ rotating text بسيط.

------------------------------------
11) ملاحظات ختامية للبناء من الصفر
------------------------------------

- مع أن الموقع Static، إلا أن تنظيم الأكواد يجب أن يكون على شكل "وحدات":
  - ملف JS عام (main.js) لسلوك الهيدر والـ scroll والـ theme.
