توصيف طبقة البيانات وهيكلية المحتوى (Data Layer Specification)

هذا الملف يشرح بالتفصيل كيف يتم تخزين البيانات في ملفات JS وكيفية الوصول إليها وعرضها في الموقع لضمان سهولة التحديث والتطوير.

1. الفلسفة البرمجية (The Approach)
نعتمد أسلوب "Data-Driven Rendering" حيث يتم فصل المحتوى (البيانات) تماماً عن القوالب (HTML). يتم تخزين البيانات في ملفات JavaScript مستقلة ككائنات (Objects) ومصفوفات (Arrays).

2. هيكلية ملف المشاريع (portfolio-data.js)
كل مشروع يتم تمثيله ككائن داخل مصفوفة أو كائن رئيسي مفتاحه هو المعرف الفريد (ID):

{
  id: "project-unique-id",
  title: "اسم المشروع",
  category: "web | mobile | desktop",
  thumbnail: "رابط الصورة المصغرة",
  description: "وصف قصير يظهر في الشبكة الرئيسية",
  fullDescription: "وصف تفصيلي يظهر في صفحة المشروع",
  technologies: ["React", "Node.js", "MongoDB"],
  client: "اسم العميل",
  projectDate: "تاريخ الإنجاز",
  projectUrl: "رابط المعاينة الحية",
  gallery: [
    "assets/img/p1.jpg",
    "assets/img/p2.jpg"
  ],
  features: [
    "ميزة 1: شرح بسيط",
    "ميزة 2: شرح بسيط"
  ]
}

3. هيكلية ملف الخدمات (services-data.js)
يحتوي على تفاصيل الخدمات المقدمة:

{
  id: "service-id",
  title: "اسم الخدمة",
  icon: "اسم أيقونة Bootstrap",
  shortDescription: "وصف يظهر في الصفحة الرئيسية",
  longDescription: "شرح معمق لآلية العمل في صفحة التفاصيل",
  features: ["دعم فني", "تطوير مستمر", "أمان عالٍ"],
  benefits: ["زيادة المبيعات", "تحسين تجربة المستخدم"],
  image: "صورة توضيحية للخدمة"
}

4. هيكلية ملف الشهادات (testimonials-data.js)
يحتوي على آراء العملاء:

{
  id: "testimonial-id",
  name: "اسم العميل",
  role: "المسمى الوظيفي",
  company: "اسم الشركة",
  image: "صورة العميل",
  comment: "نص الشهادة المكتوب",
  rating: 5 // عدد النجوم من 1 إلى 5
}

5. آلية جلب البيانات (Data Fetching Logic)

أ. في الصفحة الرئيسية (Home Page):
- يتم استخدام دالة Loop تمر على مصفوفة البيانات (مثلاً portfolioArray).
- يتم استنساخ قالب HTML (Template) لكل عنصر وتعبئته بالبيانات.
- يتم تحديد عدد العناصر (مثلاً عرض آخر 6 مشاريع فقط).

ب. في صفحة التفاصيل (Details Page):
- يتم قراءة الـ ID من رابط الصفحة (URL Parameters) باستخدام `URLSearchParams`.
- يتم البحث في ملف البيانات عن الكائن الذي يطابق هذا الـ ID.
- إذا وجد، يتم حقن البيانات في العناصر المخصصة (العنوان، الصور، الوصف).
- إذا لم يوجد، يتم تحويل المستخدم لصفحة 404 أو الصفحة الرئيسية.

6. المميزات والفوائد
- سهولة التعديل: لتغيير وصف مشروع، لا تحتاج للبحث في ملفات HTML، فقط عدل ملف الـ JS المخصص.
- الأداء: يتم تحميل البيانات محلياً وبسرعة فائقة دون الحاجة لانتظار استجابة من قاعدة بيانات خارجية.
- الجاهزية: هذه الهيكلية تجعل الموقع جاهزاً تماماً للتحول إلى React أو Vue أو أي إطار عمل حديث بمجرد تغيير طريقة الرندر (Rendering).

7. قواعد التسمية (Naming Conventions)
- الصور: يفضل وضع الصور في مجلدات مقسمة (img/portfolio/project-id/).
- الـ IDs: يجب أن تكون باللغة الإنجليزية وبدون مسافات (مثلاً smart-ship-system).
- اللغات: البيانات تدعم إضافة حقول للترجمة مستقبلاً (مثلاً title_ar, title_en).
