نظرة عامة على الصفحة

فهم الفكرة الكاملة قبل البناء

💡
الفكرة الأساسية: بدلاً من عرض مشاريع كصور فقط في شبكة رتيبة 3×3، نستخدم Bento Grid — شبكة ديناميكية حيث كل مشروع يأخذ مساحة مختلفة حسب أهميته، مع بطاقات تكشف التفاصيل عند الـ Hover.
🗂️ المكونات الرئيسية للصفحة
الصفحة تتكون من 5 أجزاء رئيسية متسلسلة من الأعلى للأسفل:
  1. Header: اسم المطور + وصف + اسم الصفحة
  2. Skill Strip: شريط يعرض التقنيات المستخدمة كـ Pills
  3. Filter Bar: أزرار الفلترة (ALL / WEB / MOBILE...)
  4. Bento Grid: شبكة المشاريع الديناميكية
  5. Stats Bar: إحصائيات الإجمالية في الأسفل

نظام الشبكة — Bento Grid

12 عموداً بارتفاعات متغيرة

📐
لماذا 12 عموداً؟ العدد 12 قابل للقسمة على 2 و3 و4 و6، مما يتيح تقسيمات مرنة: نصف الشبكة = 6 أعمدة، ثلث = 4، ربع = 3، سدس = 2.
CSS Grid
.grid {
  display: grid;

  /* 12 عمود متساوي */
  grid-template-columns: repeat(12, 1fr);

  /* ارتفاع كل صف = 100px */
  grid-auto-rows: 100px;

  /* المسافة بين الكاردات */
  gap: 8px;

  max-width: 860px;
  margin: 0 auto;
}

مقاسات الكاردات

كلاسات span للعرض وspan للارتفاع

الكلاس CSS المولّد العرض بالأعمدة الاستخدام
s2 grid-column: span 2 2 من 12 = صغير جداً أيقونات / widgets صغيرة
s3 grid-column: span 3 3 من 12 = ربع الشبكة معظم المشاريع الصغيرة
s4 grid-column: span 4 4 من 12 = ثلث الشبكة مشاريع متوسطة
s5 grid-column: span 5 5 من 12 = أقل من نصف مشاريع بارزة
s7 grid-column: span 7 7 من 12 = أكثر من نصف المشروع الرئيسي (featured)
r2 grid-row: span 2 ارتفاع مزدوج = 200px + gap للمشاريع المهمة
r3 grid-row: span 3 ارتفاع ثلاثي = 300px + gaps بطل الصفحة
CSS Classes
/* العرض (columns) */
.s2 { grid-column: span 2 }
.s3 { grid-column: span 3 }
.s4 { grid-column: span 4 }
.s5 { grid-column: span 5 }
.s6 { grid-column: span 6 }
.s7 { grid-column: span 7 }

/* الارتفاع (rows) */
.r2 { grid-row: span 2 }
.r3 { grid-row: span 3 }

/* مثال كارد كبير: s7 r2 */
/* عرض: 7/12 = 58%  |  ارتفاع: 2×100+8 = 208px */

تشريح الكارد — طبقة بطبقة

كل كارد يتكون من 5 طبقات متراكبة

🗂️
كل كارد هو position: relative يحتوي على طبقات بـ position: absolute. الترتيب من الأسفل للأعلى: الخلفية → Glow → الأيقونة → النوع → المحتوى الثابت → الـ Overlay → الشريط.
HTML — بنية الكارد الكاملة
<div class="card s7 r2 g-blue" data-c="web">

  <!-- طبقة 1: توهج الخلفية (يظهر عند hover) -->
  <div class="card-glow"></div>

  <!-- طبقة 2: أيقونة نوع المشروع (أعلى اليمين) -->
  <div class="card-icon-wrap blue">
    <i class="ti ti-layout-dashboard"></i>
  </div>

  <!-- طبقة 3: نص نوع المشروع (أعلى اليسار) -->
  <div class="card-type">Web App</div>

  <!-- طبقة 4: عنوان + وصف قصير (أسفل الكارد - دائم) -->
  <div class="card-body">
    <div class="card-title">IMA Management Dashboard</div>
    <div class="card-meta">Full-stack admin · Charts</div>
  </div>

  <!-- طبقة 5: Overlay كامل (يظهر عند hover) -->
  <div class="card-overlay">
    <div class="ov-title">IMA Management Dashboard</div>
    <div class="ov-desc">وصف مفصل للمشروع...</div>
    <div class="ov-tags">

  <!-- طبقة 6: شريط اللون أسفل الكارد -->

الحالةما يحدثCSS المسؤول
Default يظهر العنوان والـ Meta فقط، الحدود رمادية داكنة .card-body visible, .card-overlay { opacity: 0 }
Hover يرتفع الكارد 2px، الحدود تصبح زرقاء، الـ Overlay يظهر مع التقنيات .card:hover → border-color, transform, .card-overlay { opacity: 1 }
Hidden عند الضغط على فلتر لا يشمل هذا المشروع يختفي .hidden { display: none !important }

نظام الفلاتر

JavaScript يربط الأزرار بالكاردات

شكل الفلاتر في الواجهة:

HTML — Filter Buttons
<div class="filters" id="filters">
  <!-- data-f = القيمة التي ستقارن بـ data-c في الكارد -->
  <button class="fb on" data-f="all">
    ALL<span class="n">10</span>
  </button>
  <button class="fb" data-f="web">
    Web App<span class="n">3</span>
  </button>
  <!-- ... باقي الأزرار -->
</div>
JavaScript — Filter Logic
const fbs   = document.querySelectorAll('.fb');
const cards = document.querySelectorAll('.card');
const cnt   = document.getElementById('cnt');

fbs.forEach(btn => {
  btn.addEventListener('click', () => {

    // 1. أزل كلاس .on من جميع الأزرار
    fbs.forEach(b => b.classList.remove('on'));

    // 2. أضف .on للزر المضغوط
    btn.classList.add('on');

    // 3. اقرأ قيمة الفلتر
    const f = btn.dataset.f;

    // 4. مرر كل الكاردات
    let v = 0;
    cards.forEach(card => {
      // إذا "all" أو نوع الكارد يطابق الفلتر
      const show = f === 'all' || card.dataset.c === f;
      card.classList.toggle('hidden', !show);
      if (show) v++;
    });

    // 5. حدّث العداد
    cnt.textContent = v;
  });
});
🔑
العلاقة الأساسية: زر الفلتر يحمل data-f="web"، والكارد يحمل data-c="web". عند الضغط على الزر، يقارن JS القيمتين ويُخفي أو يُظهر الكارد.

خطوات البناء من الصفر

دليل تسلسلي خطوة بخطوة

  1. إعداد CSS Variables والخلفية
    أول خطوة: ضع كل الألوان كـ CSS Variables في :root. ثم اضبط background: #080808 على الـ wrapper الرئيسي.
  2. بناء Grid Container
    اكتب display:grid; grid-template-columns:repeat(12,1fr); grid-auto-rows:100px; gap:8px. هذه هي نواة التصميم كله.
  3. إنشاء كلاسات s2 → s7 و r2 و r3
    هذه الكلاسات تتحكم في حجم كل كارد. .s7 { grid-column: span 7 } — المشروع الرئيسي يأخذ 7 من 12.
  4. بناء الكارد الأساسي بطبقاته
    ابدأ بـ position:relative; overflow:hidden على الكارد. ثم أضف طبقة Glow، الأيقونة، النص الدائم، والـ Overlay المخفي.
  5. إضافة تأثيرات Hover
    على .card:hover أضف: تغيير لون الحدود، رفع الكارد 2px، وإظهار الـ Overlay بـ opacity: 1.
  6. بناء نظام الفلاتر
    أضف data-c على كل كارد وdata-f على كل زر. كتابة JS يقارن القيمتين ويضيف/يزيل كلاس .hidden.
  7. إضافة Responsive Design
    في الشاشات الصغيرة (max-width: 640px) قلّص الشبكة لـ 2 أعمدة فقط واجعل كل الكاردات span 2.
