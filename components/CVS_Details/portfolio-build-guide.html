🎨 CSS Variables — نظام الألوان
كل الألوان تُعرَّف مرة واحدة في :root. لتغيير الـ theme كله تغير 3 قيم فقط.
CSS — :root
:root {
  /* ── ألوان التطبيق الأصلية ── */
  --accent:  #0563bb;   /* اللون الرئيسي — ثابت دائماً */
  --bg:       #080808;   /* الخلفية الأعمق */
  --surface:  #1a1a1a;   /* سطح الكاردات */
  --text:     #ffffff;
  --text2:    #b0b0b0;

  /* ── أبعاد المجسم 3D ── */
  --box-w: 360px;
  --box-h: 500px;
  --box-d: 60px;   /* العمق — هذا يتحكم بسُمك الحواف */
}

/* موبايل: الـ variables تتغير فقط */
@media (max-width: 768px) {
  :root {
    --box-w: 300px;

  }

المجسم 3D — CSS Cuboid
مستطيل حقيقي بـ 6 وجوه. لا Three.js، لا Canvas — فقط CSS transforms.
📐
المبدأ الرياضي: كل الوجوه الستة تبعد d/2 عن مركز المجسم. الوجه الأمامي عند z=0، الحواف تُحرَّك للخلف بـ translateZ(-d/2) بعد الدوران.
الوجهTransformالملاحظة
FronttranslateZ(+d/2)يبرز للأمام من المركز
BackrotateY(180°) translateZ(+d/2)مقلوب ومدفوع للخلف
RighttranslateX(w) rotateY(-90°) translateZ(-d/2)origin: left center
LeftrotateY(90°) translateZ(-d/2)origin: left center
ToprotateX(90°) translateZ(-d/2)origin: center top
BottomtranslateY(h) rotateX(-90°) translateZ(-d/2)origin: center top
⚠️
القاعدة الذهبية: translateZ دائماً يأتي بعد الـ rotate — لأن بعد الدوران يتغير اتجاه المحور Z للعنصر. قبله → يتحرك في الفضاء العالمي. بعده → يتحرك في فضاء العنصر المحلي.
CSS — 6 Faces
/* الوجه الأمامي */
.card-front {
  width: var(--box-w); height: var(--box-h);
  top: 0; left: 0;
  transform: translateZ(calc(var(--box-d) / 2));
}

/* الحافة اليمنى */
.card-side-right {
  width: var(--box-d); height: var(--box-h);
  top: 0; left: calc(var(--box-w) + var(--box-d)/2 * -1 + var(--box-d)/2);
  transform-origin: left center;
  transform: translateX(var(--box-w)) rotateY(-90deg) translateZ(calc(var(--box-d) / -2));
}

نظام الـ Tilt — الماوس يقود المجسم
المجسم يتتبع الماوس من أي مكان على الصفحة بـ lerp ناعم.
JavaScript — Global Mouse Tilt
// 1. تتبع الماوس عالمياً
let mouseX = window.innerWidth / 2;
let mouseY = window.innerHeight / 2;
document.addEventListener('mousemove', e => {
  mouseX = e.clientX; mouseY = e.clientY;
});

// 2. حلقة rAF — تحسب الـ tilt كل frame
let interpX = 0, interpY = 0;
let targetX = 0, targetY = 0;

function animTilt() {
  const rect = card.getBoundingClientRect();
  const dx = (mouseX - rect.left - rect.width/2) / window.innerWidth;
  const dy = (mouseY - rect.top - rect.height/2) / window.innerHeight;

  // Sharpness: أقوى قريباً، أضعف بعيداً — لكن لا يتوقف
  const sharp = 1.8 / (1 + Math.sqrt(dx*dx+dy*dy) * 2.2);
  const amp = 0.35 + sharp * 0.65;

  targetY =  dx * 38 * amp;  // ±38° أفقياً
  targetX = -dy * 28 * amp;  // ±28° عمودياً

  // Lerp: 9% من المسافة كل frame = ناعم جداً
  interpX += (targetX - interpX) * 0.09;
  interpY += (targetY - interpY) * 0.09;

  card.style.transform = `
    perspective(1100px)
    rotateX(${interpX}deg)
    rotateY(${interpY}deg)
  `;
  requestAnimationFrame(animTilt);
}
animTilt();

Drag to Rotate — السحب يدور المجسم
ضغط + سحب = دوران حر. الإفلات = inertia ثم snap-back للـ tilt mode.
🔑
المشكلة الكبيرة المحلولة: dragMode يجب أن يكون في الـ outer scope حتى تستطيع animCardTilt قراءته. إذا كان داخل IIFE فقط → لن تتوقف عن الكتابة على الـ transform.
  1. mousedown → dragMode = true
    نسجل نقطة البداية ونوقف الـ tilt
  2. mousemove → نحسب dx/dy ونجمعها في dragRotX/Y
    الـ velocity يُحدَّث كل frame للـ inertia لاحقاً
  3. mouseup → dragDecay = true
    Inertia تبدأ: velocity * 0.92 كل frame
  4. velocity < 0.08 → snap-back
    dragRotX *= (1 - 0.04) حتى < 0.3° → dragMode = false → tilt يعود

Projects Flip Cards — 3 طبقات
كل كارد مشروع = proj-scene → proj-reveal-wrap → proj-card. كل طبقة مسؤولة عن شيء.
📦.proj-sceneperspective container فقط — لا transform، لا preserve-3d
✨.proj-reveal-wrapالـ reveal animation (translateY + blur عند scroll)
🔄.proj-cardpreserve-3d + CSS hover flip = rotateY(180deg)
🖼️.proj-face.proj-front/backbackface-visibility: hidden — يُخفي الوجه غير المرئي

الخطأ الكلاسيكي: إذا وضعت preserve-3d على الـ scene أو الـ wrap → الـ back face سيظهر أسفل الكارد عند الـ hover. الحل: preserve-3d على .proj-card فقط وليس أي parent آخر.

Skills Slide — الإزاحة من الأطراف
كل كارد skill يطير من الجهة التي هو فيها عند الوصول للـ section.
JS — getDirection + slideIn
function getDirection(card) {
  const gridCX = grid.getBoundingClientRect().left + grid.offsetWidth/2;
  const cardCX = card.getBoundingClientRect().left + card.offsetWidth/2;
  return cardCX <= gridCX ? 'left' : 'right';
}

function slideIn() {
  cards.forEach((card, idx) => {
    const offset = getDirection(card) === 'left' ? '-110vw' : '110vw';
    const delay = idx * 75; // stagger 75ms

    card.style.transform = `translateX(${offset})`;
    void card.offsetWidth; // reflow

    card.style.transition = `
      transform .75s ${delay}ms cubic-bezier(.22,1,.36,1)
    `;
    card.style.transform = 'translateX(0)';
  });
}

Bi-directional Reveal — يختفي عكساً
كل عنصر له 3 حالات: hidden (أسفل) / visible (مكانه) / out (أعلى). يعيد الـ animation عند كل scroll.
JS — Registry System
const animDefs = {
  'section-title': {
    hidden:  { opacity:'0', transform:'translateY(32px)' },
    visible: { opacity:'1', transform:'translateY(0)',
               transition:'opacity .75s ease, transform .75s ease' },
    out:     { opacity:'0', transform:'translateY(-20px)',
               transition:'opacity .4s ease, transform .4s ease' }
  }
  // ... باقي العناصر
};

function scanElements() {
  registry.forEach(entry => {
    const r = entry.el.getBoundingClientRect();
    const inView  = r.top < vh * 0.88 && r.bottom > 0;
    const aboveVP = r.bottom < -60;  // فوق الشاشة
    const belowVP = r.top > vh + 60;  // أسفل الشاشة

    if (inView)  enterEl(entry);
    if (aboveVP) exitEl(entry, 'up');   // خرج للأعلى
    if (belowVP) exitEl(entry, 'down'); // خرج للأسفل
  });
}
